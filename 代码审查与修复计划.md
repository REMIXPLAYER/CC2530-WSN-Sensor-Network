## 一、审查概述

### 1.1 总体评价

整体架构清晰（信标发现 + P2P 数据 + 广播指令 + 传感器 ACK），README/UPGRADE 文档完整度较高。但代码实现中存在**多个严重的业务正确性和设计缺陷**，尤其集中在：

- **多传感器部署的防冲突**：CSMA-CA 随机种子硬编码，同类型节点退避序列相同导致持续冲突
- **DHT11 驱动时序**：释放总线未切输入模式，校验失败时 LCD 显示未校验数据
- **接收节点串口竞态**：ISR 与主循环对 `cmdBuf` 的窄窗口竞态
- **文档与代码不一致**：短地址派生死代码、缓冲区大小、NODE_TYPE 等

### 1.2 业务需求澄清（用户确认）

| 节点 | 职责 | D6 灯期望 | D7 灯期望 |
|------|------|-----------|-----------|
| **DHT11 传感器** | 采集温湿度 → 转发给接收节点；接收 LED 开关指令；接收数据采集开关指令（STATUS=1/0） | **发送数据成功时短闪 50ms**（D6 闪烁=正在发送）；STATUS=0 或未注册时不发送自然不闪 | LED 指令控制（LED=1 亮 / LED=0 灭） |
| **光敏传感器** | 采集光照 → 转发；接收 LED/STATUS 指令 | 同上 | 同上 |
| **接收节点** | 透传数据到 PC；广播信标/指令；**LCD 显示存活状态**（不显示接收数据） | **发送（广播）数据时短闪 50ms**（D6 闪烁=正在广播信标/指令） | LED 指令控制（@ 亮 / ! 灭） |

**关键澄清（用户确认）**：
- **D6 灯**：闪烁意味着节点正在发送数据（传感器发送采集数据 / 接收节点广播信标或指令）
- **D7 灯**：由 LED 指令控制亮灭（传感器：LED=1/0；接收节点：@/!）
- **STATUS 指令**：控制传感器的数据采集发送开关（STATUS=1 发送 / STATUS=0 暂停），STATUS=0 时传感器不发送数据，D6 自然不闪
- **接收指令的副作用**：传感器接收指令本身可能影响灯状态（如 LED 指令改变 D7），这是正常行为

**LCD 接线**：LCD 屏与 PC 共用 UART0（用户确认）。`lcd_dis()` 的 `{TYPE=}` `{MAC=}` 会被 PC 收到，需 PC 网关过滤。

```c

    basicRfReceiveOff();
    basicRfConfig.ackRequest = FALSE;
    basicRfSendPacket(0xFFFF, (uint8*)cmd, strlen(cmd) + 1);  // 读 cmd
    ...
```

**竞态触发路径**：

```
主循环: cmdReady=0 → broadcastCmd(cmdBuf) → strlen(cmdBuf) → basicRfBuildMpdu(cmdBuf) → ...
                                              ↑
                              ISR 在此刻写入 cmdBuf（PC 发新 {指令}）
                                              ↓
        basicRfBuildMpdu 内部 for 循环读到部分新数据 → 发送出"半新半旧"指令
```

**触发条件**：PC 在主循环消费上一条指令的 `strlen` + `basicRfBuildMpdu` 期间（微秒级）发送新指令。概率低但理论存在，且 `halMcuWaitMs(50)` 期间 ISR 写入的 `cmdBuf` 内容会保留到下一轮（无害），**真正的风险在 strlen/basicRfBuildMpdu 窗口**。

## 五、逐个问题修复计划

### P0-1 CSMA-CA 随机种子未与 MAC 绑定

**根因**：`getRandomDelay` 的 `static seed` 硬编码（DHT11=0x4103，光敏=0x4101），同类型节点种子相同 → 退避序列相同 → CSMA-CA 退避失效 → 持续冲突。

**影响**：部署 2 台以上同类型传感器时，CSMA-CA 失效，P2P 数据传输持续冲突，丢包率飙升。

**修复方案**：种子从本机 MAC 地址派生（`macAddr[0..1]` 组合），在 `readMacAddress` 后初始化。

**修复 patch（DHT11 `main.c`）**：

```c
// === 修复前（L51-57）===
unsigned int getRandomDelay(void) {
    static unsigned int seed = 0x4103;
    seed = (seed * 1103515245 + 12345) & 0x7FFF;
    return seed;
}

// === 修复后 ===
static uint16 randSeed = 0;   // 从 MAC 派生，确保每个节点退避序列不同

void rand_init(uint8* macAddr) {
    randSeed = ((uint16)macAddr[0] << 8) | macAddr[1];
    if (randSeed == 0) randSeed = 0x4101;   // 避免 0 种子导致退避序列退化
}

unsigned int getRandomDelay(void) {
    randSeed = (randSeed * 1103515245 + 12345) & 0x7FFF;
    return randSeed;
}
```

**调用点（DHT11 `main.c` rfSendData 内，L133 后）**：

```c
readMacAddress(macAddr);
rand_init(macAddr);              // 新增：初始化随机种子
sprintf(myMacStr, ...);
```

**光敏节点同样修复**（删除 L144 的 `seed = 0x4101`，改用 `rand_init`）。

**验证方法**：部署 2 台 DHT11 节点，观察 `csmaCaSendPacket` 重试次数应明显下降，无持续冲突。

### P0-3 DHT11 释放总线后未切输入模式

**根因**：[DHT11 L397-399](file:///c:/Users/MI/桌面/workbench/CC2530-WSN-Sensor-Network/4.1-P2P-DHT11/main.c#L397-L399) 主机拉低 18ms 后 `COM_SET`（`P0_5=1`），但引脚仍是 `PIN_OUT` 输出模式。DHT11 协议要求主机释放总线（切输入 + 上拉）后，DHT11 才能拉低数据线应答。

**现状**：依赖 8051 准双向口写 1 后能被外部拉低的特性"凑巧"工作，时序不稳定，存在驱动管脚风险。

**修复方案**：`COM_SET` 后切输入模式，让总线由上拉电阻拉高，DHT11 可拉低应答。

**修复 patch（DHT11 `main.c` L397-399）**：

```c
// === 修复前 ===
COM_CLR;
halMcuWaitMs(18);
COM_SET;

// === 修复后 ===
COM_OUT;                // 确保输出模式
COM_CLR;                // 主机拉低 18ms（起始信号）
halMcuWaitMs(18);
COM_IN;                 // 切输入模式，释放总线（由上拉电阻拉高）
// 此时 DHT11 会在 20-40us 后拉低 80us 作为应答
```

> ⚠️ **硬件前提**：`COM_IN` 后 P0_5 为高阻输入，需确保硬件有外部上拉电阻（或启用 CC2530 内部上拉 `P0INP &= ~0x20`）。若硬件无上拉，保留原准双向口方案但加注释说明。

### P0-5 接收节点 `cmdBuf` 窄窗口竞态

**根因**：`cmdBuf` 是 ISR 与主循环共享缓冲区，主循环 `broadcastCmd(cmdBuf)` 直接传指针给 `basicRfSendPacket`，在 `strlen(cmd)` 和 `basicRfBuildMpdu` 拷贝 `cmdBuf` 到 `txMpdu` 期间（微秒级），ISR 若写入新指令，会发送出"半新半旧"的混合指令。

**修复方案**：`broadcastCmd` 入口关中断，将 `cmdBuf` 复制到本地缓冲区，再开中断发送本地缓冲区。

**修复 patch（接收节点 `main.c` L26-36）**：

```c

    basicRfReceiveOff();
    basicRfConfig.ackRequest = FALSE;
    basicRfSendPacket(0xFFFF, (uint8*)cmd, strlen(cmd) + 1);
    basicRfConfig.ackRequest = TRUE;
    basicRfReceiveOn();
    D6 = 0;
    halMcuWaitMs(50);
    D6 = 1;
}

// === 修复后 ===
// 头部需新增 #include "hal_int.h"
void broadcastCmd(char* cmd) {
    char localBuf[CMD_BUF_SIZE];
    uint16 intState;
    uint8 len;

    // 关中断复制 cmdBuf 到本地，避免 ISR 在发送期间覆盖
    intState = halIntLock();
    len = strlen(cmd);
    if (len >= CMD_BUF_SIZE) len = CMD_BUF_SIZE - 1;
    memcpy(localBuf, cmd, len);
    localBuf[len] = '\0';
    halIntUnlock(intState);

    basicRfReceiveOff();
    basicRfConfig.ackRequest = FALSE;
    basicRfSendPacket(0xFFFF, (uint8*)localBuf, strlen(localBuf) + 1);
    basicRfConfig.ackRequest = TRUE;
    basicRfReceiveOn();
    D6 = 0;
    halMcuWaitMs(50);
    D6 = 1;
}
```

